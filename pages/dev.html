Professional

Since I began working as a Software Developer, I have completed a number of assignments in a variety of languages.

I designed and built a software installation script in Powershell for my first asignment. The script functioned is launched by double-clicking on an icon within the installation CD. It began by disabling a number of running Microsoft services and detaching multiple databases from MSSQL Server. Then it copied files over from the CD into specific directories on the computer. Lastly, the script restarted the computer, reattched the databases and enabled the services.

Next I contributed to a ASP.NET Razor Pages web site. I created stories and pushed code to an Azure DevOps repository. On this project I used C#, HTML, CSS, Javascript, and Regex. My contributions added additional front-end and back-end functionalities.

Recently, I studied, designed, and delivered an enterprise-level software design proposal. The project has since transitioned into development which is ongoing. I feel like it is an especially valuable experience to work as a developer on a large software project I helped to design. The product is an instantiable containerized software. Each instance is able to accept unique data streams from a source via TCP/IP connection The data streams are then parsed into messages and then published to a message broker (like ActiveMQ). An instance is also able to recieve messages as a subscriber, reconstruct the messages to a data stream and send the stream to a recipient system via TCP/IP. The intent is to allow for a third-party to create subscribers to recieve standardized messages intended for a variety of recipients. A goal of the enterprise serice is to remain tecnology stack independent. One part of this is to use an API wrapper for the message broker that allows for using one standard for a number of messaging services. There is essentially two main development efforts. The first portion of development will focus on creating the message passing infrastructure to accept and send data streams, create messages, and interact with the message broker. The second effort is to work with each recipient system's unique message formats to accurately parse data streams into messages and back, while maintaining data integrity.
